Chapter 10: Testing and Debugging

There are different types of errors:

System Error: a problem with the system or external devices.

Programmer Error: the program has incorrect syntax or faulty logic.

User Error: the user has entered data incorrectly which the program is unable to handle. Programmer should anticipate user errors and prepare the program to handle them.

Exceptions:

An error that produces a return value that can then be used by the program to deal with the error.

Stack Trace:

A sequence of functions or methods calls that lead to the point where the error occurred.

Warnings:

Can occur if there's an error in the code but it isn't enough to make the program crash.

Strict Mode:

It produces more exceptions and warnings. It allows to spot errors earlier in the development face. To write in strict mode write the following line at the top of the JS file: 'use strict'.

Debugging in the Browser

Find the point in the code where errors occur. Use breakpoints that halt the progress of the code and allow you to see the value of variables at that point in the program.

There are several options for debugging:

Alerts:the 'alert()' method allows to stop the program to check variables' values. Use currently discouraged.

Using the console:the 'console' object provides several methods for logging information and debugging.

console.log() => can be used to log the value of variables at different points in the program but it doesn't stop the program from running.

console.trace() => will log an interactive stack trace in the console. Shows the functions that were called in thelead up to an exception occurring.

Debugging Tools

The 'debugger' keyword creates a breakpoint in the code that pauses its execution and lets you see where the program is currently up to. You can see variable values by hovering over them with the mouse. It is important to remove the keywords before shipping code because they will make the code stop.

Error Objects

These objects can be created in the code using a constructor function.

example: const error = new Error();

Takes a parameter that's used as the error message.

There are seven more error objects used for specific errors:

EvalError => currently not being used. It was used to identify errors when using the 'eval()' function.

RangeError => thrown when a number is outside allowable range of values.

ReferenceError => when a reference is made to an item that doesn't exist.

SyntaxError => there's an error in the code's syntax.

TypeError => error in the type of value used.

URIError => there's a problem encoding or decoding the URI

InternalError => non-standard error. Thrown when an error occurs in the JS engine.

'error' objects have a number of properties:

name => returns the name of the error constructor function used as a string.

message => returns a description of the error. Should be provided as an argument to the 'Error' constructor function.

stack => returns a stack trace for that error. Not safe to use in production sites.

Throwing Exceptions

You can throw your own exceptions using the "throw" statement. Use "throw" with an "error" object.

Exception Handling

When an exception occurs, the program stops running. We can handle this situation by catching the error.

If we think that a piece of code will result in an exception, we can wrap it in a 'try' block. the code will run as normal but if an error occurs, we can pass the error object to a 'catch' block where we can deal with the error appropriately.

A 'finally' block can be added after a 'catch' block. It'll always be executed after the 'try' or 'catch' block, even if an exception doesn't occur.

Tests

Writing good tests helps strengthen the code and allows for errors to be caught early.

Test-driven Development (TDD)

It means to write tests before writing any actual code.

Workflow:

  1. Write tests (that initially fail)
  2. Write code to pass the tests
  3. Refactor the code
  4. Test refactored code
  5. Write more tests for new features

Jest

It is a TDD framework created by Facebook. It provides helper methods for common test assertions.

Function test() takes two parameters. First parameter is a string that describes what we are testing. The second parameter is an anonymous function that contains a function called 'expect()' => This takes the function we are testing as an argument. Returns an expectation object. The expectation object has methods called 'matchers'. Some examples of matcher methods are: 'toBe()' and 'toEqual()'
